GitTracker (2015)

This is a tool I built as an experiment while learning javascript. It tracks when a user commits code and then counts the number of commits per day. There wasn't really a purpose to it, other than teaching myself javascript.

The Github users were chosen from this list, which I found by Googling "active github users." This is the first thing that I've built with Javascript and I just wanted to challenge myself in a fun way. Don't look too much into it.

So how'd I do it?

(tl;dr view the Github repo here)

  1. I created an ajax call to the Github API, selecting a specific user's data.
  2. If I successfully connected, I created a function that stored the returned data.
  3. I used a for() loop that would loop through every result
  4. I assigned the API value: "pushed_at" to a "pushed_at" variable
  5. I eliminated any text in the string that came after the date and assigned it to variable "datePushed". "pushed_at" was updated any time a commit was pushed to any of the repository's branches.
  6. I created a variable named "count", to store the number of times the user pushes code on a each date.
  7. I nested another for() loop that would run through the length of the current data stored in the "result" paremeter.
  8. I assigned the API value "pushed_at" to "Match_pushed_at" variable.
  9. I again eliminated any text in the string that came after the date and assigned the result to the variable "datePushed"e;
  10. I used an if() statement to determine whether the value of loop 1 was equal to the value of loop 2. If the value was equal, it meant the user pushed code on that date more than once. I would then increment the "count" variable.
  11. After running through every piece of data stored in the "result" parameter, an if() statement would run inside the first loop.
  12. This statement checked if the "datePushed" had already been stored in the array named "arr1" (more on that later). If so, it was a duplicate. I needed to count duplicates to sum the number of pushes on the given date, but I didn't want each date to display to the user.
  13. Therefore, duplicates were not added to the array named "match". Later in the script, I append text to the #pushList element ID based on the values stored in "match".
  14. Further, if the value in "datePushed" was not a duplicate, I would store the "count" value in another array named, "arr2".
  15. Next, every single "datePushed" value in the loop was added to the "arr1" array so that the above if() statement could properly cross-check whether any future "datePushed" values were duplicates.
  16. I then created yet another for() loop which looped through the "match" array and appended the relevant HTML to the HTML element with the "pushList1" ID.
  17. Finally, I used an if() statement to use plural language if the user pushed code more than once on a given date.
  18. I repeated this process for a total of three different Github users, generating the results of each user in their own column.
  19. That's it!

I'm certain there is a more efficient way for me to structure this script, but I wrote this after just starting to learn Javascript. In fact, if you look carefully, you'll see that each user usually pushed code only once per date (though they occassionally pushed more often). This has led me to speculate that there's an error in my code somewhere. I may go back one day to find the bug (if there is one) and optimize the script. For now though, that's it!
